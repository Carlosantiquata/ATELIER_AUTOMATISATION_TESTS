🎯 Atelier — Automatisation des tests d'API

Atelier M1 Python + Flask PythonAnywhere
📊 Dashboard ▶ Run ❤️ Health ⬇ Export
1) Objectifs & livrable final
ObjectifTester automatiquement une API (contrat + robustesse + QoS) et publier un dashboard.
LivrableUn dépôt GitHub + une app Flask sur PythonAnywhere + un historique de runs en SQLite.
ContraintesTests non destructifs, charge limitée, clés API via Secrets GitHub.
2) Scénario pédagogique
  1. Choisir une API (idéalement sans clé) et documenter le contrat.
  2. Écrire des tests "as code" : assertions sur le statut HTTP, JSON, champs et types.
  3. Ajouter de la robustesse : timeout, retry, gestion 429/5xx.
  4. Mesurer la QoS : latence (avg/p95), taux d'erreur, disponibilité.
  5. Déployer sur PythonAnywhere et exposer un dashboard lisible.
  6. Planifier l'exécution et conserver l'historique (SQLite).
⚠️ Les tests doivent être non destructifs et respecter un rythme raisonnable (1 run / 5 min, 20 requêtes max/run).
3) Architecture cible
Repo GitHub
  └── Code Python + tests
           ↓ (GitHub Actions)
PythonAnywhere (Flask)
  ├── /run        → déclenche un run
  ├── /dashboard  → résultats + historique
  └── SQLite      → stockage des runs
Structure recommandée
project/
├─ flask_app.py
├─ tester/
│  ├─ client.py    # wrapper HTTP (timeout, latence)
│  ├─ tests.py     # tests (fonctions)
│  └─ runner.py    # exécute + calcule métriques
├─ storage.py      # SQLite: save_run(), list_runs()
├─ templates/
│  └─ dashboard.html
└─ requirements.txt
4) Plan de tests attendu

A. Tests Contrat (fonctionnels)

  • HTTP 200/204 attendu
  • Content-Type JSON
  • Champs obligatoires présents
  • Types corrects (string/int/bool…)
  • Cas invalide → 400/404 attendu

B. Robustesse & QoS

  • Timeout (3s) + 1 retry max
  • Gestion 429 : backoff / attente
  • Latence : avg + p95 sur N appels
  • Taux d'erreur + disponibilité

# API Choice

Étudiant : Alexis Carlos TB
API choisie : Jikan API (HxH 2011)
URL base : https://api.jikan.moe/v4
Auth : None
Endpoints testés :

  - HTTP 200 attendu, 404 pour IDs inconnus
